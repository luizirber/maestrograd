Trabalho 2: Armazenamento temporário de ligações 0800 em espera. Individual.

 

Uma central de atendimento a clientes tem vários atendentes, mas um número muito maior de linhas telefônicas recebendo chamadas. As chamadas são colocadas em espera segundo a ordem de chegada, e atendidas quando possível. Ocorre que algumas destas chamadas vêm de longe e, neste caso, se elas ficam esperando na linha, causam uma despesa muito maior do que as chamadas que vem de perto. Uma solução alternativa seria colocar as chamadas em fila segundo a prioridade definida primeiramente pelo custo (as mais caras devem esperar menos) e secundariamente por ordem de chamada. Projete uma estrutura para modelar essa situação alternativa.

- Projete a estrutura - defina funcionalidade;

- Projete a estrutura  faça um diagrama e explique resumidamente o funcionamento;

- Implemente.

 

Sobre as estruturas a serem escolhidas:

- Deve atender ao solicitado no texto acima;

- Deve ser encadeada;

- Deve ser (em algum ponto) diferente das estruturas vistas em aula.

 

Implemente uma aplicação para simular a solução do problema em questão. Você poderá escolher o nível de sofisticação da aplicação. O importante é que resolva o problema proposto utilizando um Tipo Abstrato de Dados para armazenamento temporário das ligações em espera.

 

Para todos os trabalhos

Implemente (em C, C++ ou Pascal e derivados). Entregue listagem dos programas, um print screen da execução, e uma breve documentação (descrição de cada objeto, desenho das estruturas, etc.).

 

Possíveis Problemas

1- trabalho incompleto ou diferente do especificado;

2- erro nos algoritmos;

3-  execução inexistente ou insuficiente;

4- não usar arquivos separados para módulos independentes;

5- não respeitar as características fundamentais de TADs/Objetos;

6- incluir  em TADs  ou objetos tarefas que não lhe dizem respeito;

7- não entregar no prazo

9- trabalhos muito iguais

10- não entregar especificação dos objetos

 

 

 

Trabalho 1: Aplicação Livre Usando Filas. Individual.

 

Implemente uma aplicação que utilize filas. Você poderá escolher a aplicação, e seu nível de sofisticação. A aplicação pode ser simples. O importante é que exemplifique bem o uso de filas. Preferencialmente que utilize mais que uma (instância do TAD/objeto) fila.

 

Faça duas implementações do mesmo TAD/objeto fila: (1) uma implementação com alocação seqüencial e estática; e (2) uma implementação com alocação encadeada e dinâmica. As implementações precisam ser diferentes do visto em aula.

 

O programa principal deverá usar apenas uma das 2 implementações da fila: teste uma vez com uma, depois compile novamente para testar com a outra, SEM ALTERAR QUALQUER COMANDO NO PROGRAMA PRINCIPAL (exceto o nome do arquivo do TAD/objeto fila, caso isso seja conveniente). A fila deve implementar a fila e... só a fila mesmo.
